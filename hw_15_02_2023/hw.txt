1. отказаться от изменений в файле (файлах) находящийся в рабочем каталоге: git checkout <имя файла или .(для всех)>;
2. вернуть файл из индекса: git restore --staged <имя файла>;
3. отменить ошибочный commit: git reset HEAD~1;
4. создать новую ветку и переключится в нее: git checkout -b <имя ветки>;
5. fast-forward merge - если сливаемые ветки не содержат изменений в одном и том-же файле, т.е. указатель HEAD будет установлен на последний коммит,
non fast-forfard merge - когда слияние просто переносом указателя HEAD неврзможно;
6. Конфликт слияния возникает когда в разных ветках есть коммиты изменения одного и того же файла, для разрешения конфликта нужно вручную выбрать
какие из изменений конфликтного файла попадут в коммит слияния;
7. удалить ветку: git branch -d <имя ветки>
